Alarm System

Here we see the Spark Core! The Spark Core with Chip Antenna 1.0 uses the TI CC3000 module for Wi-Fi communications in order to connect to the Spark.io network cloud to send and receive data to and from the Particle/Spark Cloud. This enabled us to connect to the Cloud in a one-way connection and upload code through the Core, allowing events to be detected by the end data application.

Alarm System

This image shows the PIR sensor. This device is made of a pyroelectric sensor which detects levels of infrared radiation. The sensor is split in two halves so if one half sees more or less IR radiation than the other, the output will swing high or low, indicating that a motion has been detected. This information is received by the Spark Core.

Alarm System

This image shows the web page to the alarm system application. It has a button to activate and deactivate the alarm, a text area above the buttons to display when the alarm is ON/OFF or if a motion has been detected, and a text box to the right to keep a log of events.

Using a technology called Server Sent Events (SSE), a browser receives automatic updates from the Particle Cloud server via an HTTP connection, to display if the motion sensor has been triggered or not.
